QPT 1.0.7 Release Notes

NOTE 2: Windows users please see QPT-267 below before installing!

QPT 1.0.7 addresses 11 issues, described below. Release notes for the previous release are available here.

The QPT documentation index is available here.

To install and run this release of QPT:
To install → launcher -i:qpt -a:qpt-2008B -h:sbfmaps.cl.gemini.edu -p:9999
To run → launcher -a:qpt-2008B
To uninstall → launcher -a:qpt-2008B -u
As always, the first time you run QPT it will download necessary components. This may take several minutes, especially in the north. Your previous installation of QPT should continue to function normally.

New and Notable

[QPT-149] Improved sequence summary text

The sequence summary in the properties view now indicates the step type in addition to time and repeat count. This should make it a little easier to decide where to split an visit.

[SCT-336/QPT-271] Improved handling of long observations

To make it easier to deal with gigantic observations, when you create a new visit by dragging a candidate observation into the Plan Visualizer, the visit will automatically be truncated to the current "drag limit" (10 hours by default; there are menu commands to adjust this). You can then shorten/lengthen the visit normally, with less of the initial overlapping mess. Because QPT now truncates observations without telling you, there is a new QC-only warning for visits that have remaining steps (this warning doesn't appear on the published plan).

[QPT-267] - QPT now runs correcly on Windows XP

The old launcher relied on some file locking functionality that fails spectacularly on Windows, so it's now enabled only for real operating systems. Windows users will need to download the new launcher.jar before installing QPT. Please make a backup of your old one first.

Other Significant Improvements

  • [SCT-330/QPT-272] - Instrument configurations are now ordered more reasonably.
  • [SCT-356/QPT-277] - There are now forward/backward navigation buttons in the calendar control.

Significant Bug Fixes

  • [QPT-270] - Splitting a long observation would sometimes cause QPT to hang. This has been fixed.
  • [SCT-329/QPT-274] - The visualizer now indicates the displayed timezone.
  • [SCT-355/QPT-275] - Updated observing conditions (in the OT) are now seen on refresh in QPT.
  • [SCT-318/QPT-276] - The Save command is now enabled for new plans (so you don't have to select Save As).
